# 아파트 전세가율 예측 프로젝트 정리

## 1. 문제 정의

본 프로젝트는 성남시 아파트 **매매 실거래가와 전세 실거래가 데이터를 결합하여 전세가를 예측**하는 머신러닝 프로젝트이다.

초기 목표는 다음과 같았다.

- 아파트 매매가격과 전세가격 관계를 활용하여 전세가격 예측
- 시계열 기반 학습 구조 적용
- 머신러닝 모델(XGBoost)을 이용한 예측

---

# 2. 초기 문제점

초기 실험에서 다음과 같은 문제가 발견되었다.

### 2.1 데이터 부족 문제

처음에는 **동일 연도 매매–전세 데이터만 매칭**하였다.

하지만 이 방식은 다음과 같은 문제가 있었다.

- 샘플 수 약 **2,000개 수준**
- 같은 단지라도 동일 연도 거래가 존재하지 않는 경우 다수
- 모델 학습 데이터 부족

### 해결 방법

매매연도와 전세연도 차이를 **±1년까지 허용**하였다.

```
|saleYear - year| <= 1
```

이를 통해 데이터셋 크기가

```
약 2,000 → 약 8,000 rows
```

로 증가하였다.

---

### 2.2 오래된 데이터가 성능을 악화시키는 문제

처음에는 데이터를 더 늘리기 위해

```
2018 ~ 2025
```

까지 데이터를 사용하였다.

그러나 결과는 오히려 성능이 악화되었다.

이유는 다음과 같다.

- 부동산 시장은 **stationary 데이터가 아님**
- 시장 국면이 크게 변함

따라서 다음과 같이 학습 기간을 제한하였다.

```
train <= 2023
validation = 2024
test = 2025
```

---

### 2.3 모델이 단지 이름에 과도하게 의존하는 문제

Feature importance를 확인한 결과 다음과 같은 패턴이 나타났다.

```
apartmentName_***
dong_***
```

즉 모델이 사실상

> "이 단지는 평균적으로 전세가율이 이 정도"

라는 **lookup 방식으로 작동**하고 있었다.

이는 부동산 데이터에서 흔히 나타나는 현상이다.

---

### 2.4 평균 평가 지표의 한계

초기에는 다음 지표만 사용하였다.

```
MAE
MAPE
```

하지만 평균 지표만 보면 실제 모델 성능을 왜곡할 수 있다.

예시

| 실제 | 예측 | 오차 |
|----|----|----|
3억 | 3.1억 | 3%
4억 | 4.2억 | 5%
6억 | 5.9억 | 2%
2억 | 4억 | 100%

평균 MAPE는 높지만 실제로는 대부분 잘 맞춘다.

따라서 평가 지표를 다음과 같이 확장하였다.

```
MAE
RMSE
MAPE
Median MAPE
p75
p90
p95
Worst prediction 분석
```

---

# 3. 모델 개선 과정

## 3.1 타깃 변환 (Target Transformation)

전세가격 자체를 예측하는 대신

```
jeonseRatio = jeonsePrice / salePrice
```

를 예측하도록 변경하였다.

이후 예측된 비율을 매매가격과 곱하여 전세가격을 계산하였다.

```
predicted_jeonse_price = predicted_ratio * sale_price
```

### 학술적 명칭

**Ratio-based Target Engineering / Target Transformation**

### 참고 논문

공석현·오하영 (2023)  
"머신 러닝 기법을 이용한 주요 지역 매매가격 예측: 전세 제도를 중심으로"

---

## 3.2 시간 차이 피처 추가

매매와 전세가 매칭될 때 **연도 차이**가 존재할 수 있다.

이를 반영하기 위해 다음 변수를 추가하였다.

```
match_gap_year = saleYear - year
```

### 학술적 명칭

**Temporal Distance Feature**

---

## 3.3 시계열 특징 추가

같은 단지의 전세가율은 과거 값과 높은 자기상관을 가진다.

이를 반영하기 위해 다음 피처를 추가하였다.

```
last_jeonse_ratio
```

즉

```
같은 apt_id + area_group
이전 연도의 전세가율
```

을 feature로 사용하였다.

### 학술적 명칭

**Lag Feature / Autoregressive Feature**

### 참고 논문

배성완·유정석 (2018)  
"머신 러닝 방법과 시계열 분석 모형을 이용한 부동산 가격지수 예측"

---

## 3.4 데이터 누수(Data Leakage) 방지

다음 피처는 실험 중 제외하였다.

```
dong_mean_jeonse_ratio
apt_mean_jeonse_ratio
```

이유

전체 데이터셋 평균을 계산하면 **미래 데이터가 학습 feature에 유입될 위험**이 있다.

### 학술적 명칭

**Data Leakage Prevention**

---

# 4. 모델 구조

모델은 다음과 같다.

```
XGBoost Regressor
```

### 주요 파라미터

```
max_depth = 6
learning_rate = 0.05
subsample = 0.8
colsample_bytree = 0.8
min_child_weight = 10
reg_alpha = 0.1
reg_lambda = 1.0
```

---

# 5. 데이터 분할

랜덤 분할이 아닌 **시계열 기반 분할**을 사용하였다.

```
Train : <= 2023
Validation : 2024
Test : 2025
```

### 학술적 명칭

**Temporal Holdout Validation**

# 6. 평가 지표

다음 지표를 사용하였다.

```
MAE
RMSE
MAPE
Median MAPE
p75 / p90 / p95
Worst prediction 분석
```

# 7. 실험 결과

```
Test MAE: 8926
Test RMSE: 12562
Test MAPE: 11.8%
Median MAPE: 9.8%
```

오차 분포

```
p75  : 16.7%
p90  : 22.9%
p95  : 27.3%
max  : 204%
```

해석

- 절반의 예측은 **10% 이하 오차**
- 대부분은 **20% 이하 오차**
- 일부 특이 거래가 평균을 크게 증가시킴

---

# 8. Worst Prediction 분석

Worst prediction의 특징

- 고가 아파트
- 전세가율이 매우 낮은 거래
- 특정 지역 (정자동 등)

예시

```
매매가 : 168000
전세가 : 38000
전세가율 : 0.22
```

이러한 극단값이 평균 MAPE를 크게 증가시켰다.

---

# 9. 현재 프로젝트 수준 평가

현재 프로젝트는 다음 요소를 포함한다.

- 데이터 ETL 구축
- 시계열 기반 데이터 결합
- Feature Engineering
- Data Leakage 검증
- 머신러닝 모델 학습
- 분포 기반 평가

이는 단순 모델 구현을 넘어 **Applied ML 프로젝트 수준**으로 볼 수 있다.
